# AGENTS.md

Guidance for AI coding agents working in this repository.

## Repository overview

DSX Exchange is a monorepo for the DSX event bus: AsyncAPI schemas, NATS auth-callout service (Go), Helm charts, Fern docs site, and a Kind-based local evaluation framework.

## Layout

| Path | Language | Purpose |
|------|----------|---------|
| `auth-callout/` | Go | NATS auth callout service (OAuth2, mTLS, NKey, noauth) |
| `auth-callout/deploy/` | Helm | Auth-callout subchart |
| `deploy/nats-event-bus/` | Helm | Main event bus chart |
| `deploy/scripts/` | Bash | NKey generation (`generate-nkeys.sh`) |
| `schema/schema/` | YAML | AsyncAPI 3.1.0 specs (bms, power-management, nico, spiffe-exchange) |
| `docs/` | Markdown/MDX | Fern docs content (7 operator pages + generated schema pages) |
| `fern/` | YAML/TSX/CSS | Fern docs configuration and components |
| `scripts/` | Python | Schema doc generator (`generate_asyncapi_docs.py`) |
| `tools/` | Bash | `check-docs-mdx` — MDX safety linter |
| `local/` | Bash/Go | Kind-based eval environment, MQTT test client, benchmarks |
| `.github/workflows/` | YAML | GitHub Actions CI, Fern docs preview/publish |

## Build and test

```bash
make check                     # license headers + unit tests + helm lint
make test                      # unit tests only (no cluster required)
make test-e2e                  # requires Kind clusters (see local/)
make -C auth-callout test      # auth-callout unit tests
helm lint deploy/nats-event-bus
helm lint auth-callout/deploy
```

## Commit conventions

Commits follow [Conventional Commits](https://www.conventionalcommits.org/). CI enforces this via commitlint.

```
type(scope): short description
```

Allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`.

All commits must include a DCO sign-off (`git commit -s`). Semantic-release on main generates tags and changelog from commit types.

CI checks this. Run `make add-license-headers` to fix.

## Go conventions

- Go modules use vendored dependencies (`-mod=vendor`).
- `auth-callout/` has its own `go.mod`, `.golangci.yml`, and `vendor/`.
- `local/mqtt-client/` and `local/mqttbs/` are separate Go modules.

## Helm chart conventions

- The main chart is `deploy/nats-event-bus/` with `auth-callout/deploy/` as a subchart dependency.
- Values follow the `eventBus.*` namespace for bus config, `auth-callout.*` for the subchart.
- Chart validation: `helm lint` + template rendering in CI.

## Fern docs

- Config: `fern/docs.yml` with `global-theme: nvidia`.
- Docs content: `docs/` directory (Markdown and MDX).
- Schema pages: generated by `scripts/generate_asyncapi_docs.py` from AsyncAPI YAML specs.
- CI: `fern check` + `tools/check-docs-mdx` + offline link checking.
- Preview: Fern preview URLs posted as PR comments by CI.
- Do not upgrade the Fern CLI version or modify `fern/fern.config.json` version without explicit instruction.

## CI

- GitHub Actions on NV-managed runners (`linux-amd64-cpu4`).
- Triggered on push to `main` and `pull-request/[0-9]+` branches (copy-pr-bot pattern).
- `pull_request` trigger is not used — the copy-pr-bot vets external PRs before CI runs.
- Secrets (`DOCS_FERN_TOKEN`) are available because CI runs on the main repo, not forks.

## Security

- Never interpolate secrets into shell command strings — use env vars only.
- Validate all `workflow_dispatch` inputs before use.
- GHA workflows are reviewed by CodeRabbit and Greptile for CI security.
- `CODEOWNERS`: `.github/` changes require `@NVIDIA/dsx-sw-admins` review.

## AsyncAPI schemas

- Specs are in `schema/schema/{service}/{spec}.yaml` (AsyncAPI 3.1.0).
- Four specs: bms, power-management, nico, spiffe-exchange.
- BMS is the largest (~7000 lines). Message definitions should include `name:` and `title:` fields.
- Regenerate docs after spec changes: `python3 scripts/generate_asyncapi_docs.py`.

## NKey generation

- `deploy/scripts/generate-nkeys.sh` requires `nsc` and `jq`.
- Output layout: `secrets/{cluster}/nkeys/{secret-name}/{seed,pubkey}`.
- mTLS keys are always generated regardless of `eventBus.mtls.enabled`.
- Cross-cluster leaf credentials must be manually copied from CSC to each CPC.
